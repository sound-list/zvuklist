# 📂 Полный список файлов проекта

## 🎯 Главные файлы приложения

### 🌐 [index.html](index.html)
**Главный HTML файл приложения**

- Содержит: Структуру всей SPA, боковую навигацию, плеер, все 7 разделов
- Размер: ~5KB (минифицированный HTML)
- Линки: `<link href="style_spotify.css">` и `<script src="script_spotify.js">`
- Роль: Главная точка входа приложения

### 🎨 [style_spotify.css](style_spotify.css)
**Все стили приложения (CSS)**

- Содержит: 700+ строк CSS, Glass UI дизайн, адаптивный дизайн
- CSS переменные: Цвета, размеры, времена анимаций
- Брейкпоинты: 1024px (планшет), 768px (малый планшет), 480px (мобильный)
- Эффекты: Glassmorphism, анимации, переходы, gradients
- Роль: Внешний вид и дизайн всего приложения

### ⚙️ [script_spotify.js](script_spotify.js)
**Вся логика приложения (JavaScript)**

- Содержит: 700+ строк JavaScript, класс `SpotifyMusicPlayer`, 40+ методов
- Архитектура: ES6 класс с инициализацией в конструкторе
- Функциональность: Плеер, поиск, плейлисты, рекомендации, история, избранное
- localStorage: Сохранение всех данных пользователя
- Роль: Вся функциональность приложения

### 🎵 [music.json](music.json)
**Данные о треках, жанрах и плейлистах**

- Структура: 3 массива (tracks, genres, playlists)
- Треки: 10 примеров с полными метаданными
- Жанры: 6 встроенных жанров с иконками и цветами
- Плейлисты: 3 примера встроенных плейлистов
- Роль: Источник данных для приложения (вы здесь добавляете свою музыку)

## 📁 Папка с музыкой и обложками

### 🎵 [assets/music/](assets/music/)
**Папка с MP3 файлами музыки**

- Назначение: Здесь хранятся все аудиофайлы
- Формат: MP3 (рекомендуется 128-320 kbps bitrate)
- Размер файлов: Не более 5MB на трек (оптимально)
- Как добавлять:
  1. Положите MP3 в эту папку
  2. Переименуйте в track1.mp3, track2.mp3 и т.д.
  3. Обновите music.json с информацией о треке

**Примеры файлов:**
```
assets/music/
├── track1.mp3
├── track2.mp3
├── track3.mp3
├── ...
└── track10.mp3
```

### 🖼️ [assets/covers/](assets/covers/)
**Папка с обложками треков**

- Назначение: Изображения обложек для каждого трека
- Форматы: JPG, PNG, WebP (рекомендуется JPG)
- Размер: Рекомендуемый для экономии 300x300px
- Как добавлять:
  1. Положите изображения в эту папку
  2. Переименуйте в cover1.jpg, cover2.jpg и т.д.
  3. Обновите music.json с путем на обложку

**Примеры файлов:**
```
assets/covers/
├── cover1.jpg
├── cover2.jpg
├── cover3.jpg
├── ...
└── cover10.jpg
```

## 🚀 Как использовать файлы

### Для запуска приложения
1. Откройте `index.html` с Live Server или Python сервером
2. Автоматически загружается `style_spotify.css` и `script_spotify.js`
3. Автоматически загружается `music.json` с музыкой

### Для добавления музыки
1. Поместите MP3 в `assets/music/`
2. Поместите обложки в `assets/covers/`
3. Отредактируйте `music.json` с информацией о треках
4. Перезагрузите браузер (F5)

### Для кастомизации
1. Отредактируйте `style_spotify.css` для изменения дизайна
2. Отредактируйте `script_spotify.js` для изменения функций
3. Отредактируйте `index.html` для изменения структуры
4. Перезагрузите браузер (F5)

### Для решения проблем
1. Откройте консоль браузера (F12)
2. Посмотрите на ошибки
3. Обратитесь к `TROUBLESHOOTING.md`

## 💡 Полезные советы

### Совет 1: Фиксируйте основные файлы
Не считайте версией изменения в файлах старой версии (style.css, script.js).
Работайте только с новыми файлами (style_spotify.css, script_spotify.js).

### Совет 2: Резервные копии
Перед началом редактирования:
```powershell
# Скопируйте файлы на случай ошибки
Copy-Item style_spotify.css style_spotify.css.bak
Copy-Item script_spotify.js script_spotify.js.bak
```

### Совет 3: Проверка JSON
Перед редактированием music.json:
```bash
# Проверьте синтаксис
python -m json.tool music.json
```

### Совет 4: Тестирование в консоли
Откройте F12 → Console и тестируйте функции:
```javascript
console.log(player); // Вся информация о приложении
player.renderRecommendations(); // Протестировать функцию
```
